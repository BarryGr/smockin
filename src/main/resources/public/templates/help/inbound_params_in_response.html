{{ inboundParamsInResponseTitle }}
Inbound parameters and dynamic content can be included in the response body of your simulated endpoint, by using the following naming convention:

${REQ_HEAD=param_name}
${REQ_PARAM=param_name}
${PATH_VAR=param_name}
${ISO_DATE}
${ISO_DATETIME}
${UUID}
${RANDOM_NUMBER}
${RANDOM_NUMBER=numeric_range} (which can be called in 2 ways ${RANDOM_NUMBER=2 TO 5} or ${RANDOM_NUMBER=2 UNTIL 5})


To give an example, let's assume the url:

http://localhost:8001/hello?firstname=joe

To include the request parameter 'firstname' in your response you would do the following:

{ "msg" : "Hello ${REQ_PARAM=firstname}!" }

This would then return:

{ "msg" : "Hello joe!" }

In the event the request parameter firstname is not present in the request, a blank value will be returned instead:

{ "msg" : "Hello !" }


Below is an example of combining all 3 of the inbound parameter values within a response:

The endpoint setup:

Method:
GET
URL:
http://localhost:8001/greeting/:lastname
Response Body:
{ "msg" : "Greetings ${REQ_PARAM=title} ${PATH_VAR=lastname}. How are you on this ${REQ_HEAD=weather} day?" }
Calling into the endpoint:

curl -i -H "weather: sunny" -X GET http://localhost:8001/greeting/Smith?title=Mr

Output from call:

{ "msg" : "Greetings Mr Smith. How are you on this sunny day?" }


Here is an example of adding a couple of dynamic fields such as ISO_DATE and RANDOM_NUMBER to a response:

The endpoint setup:

Method:
GET
URL:
http://localhost:8001/holiday
Response Body:
{ "msg" : "The date today is ${ISO_DATE}. Only ${RANDOM_NUMBER=5 TO 15} days until your holiday!" }
Calling into the endpoint:

curl -i -X GET http://localhost:8001/holiday

Output from call (assuming the call is made on the 1st January 2017):

{ "msg" : "The date today is 2017/01/01. Only 8 days until your holiday!" }

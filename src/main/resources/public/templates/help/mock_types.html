In Sequence


An endpoint using Sequenced Responses, stores a list of multiple responses. The first call to the endpoint results with the first response in the list being returned back to the call. Subsequent calls sequentially pick up responses from the list, until the end of the list is reached and returns to the beginning again.

To give an example, you could have the endpoint http://localhost:8001/weather set up to return 3 sequential responses:


200 OK
{ "weather" : "sunny", "celsius" : 32 }

200 OK
{ "weather" : "cloudy", "celsius" : 25 }

200 OK
{ "weather" : "rainy", "celsius" : 22 }



To take make things interesting, you could then add 1 more response so as mimic the weather service failing ungracefully with an error 500 on the 4th call:


200 OK
{ "weather" : "sunny", "celsius" : 32 }

200 OK
{ "weather" : "cloudy", "celsius" : 25 }

200 OK
{ "weather" : "rainy", "celsius" : 22 }

500 INTERNAL SERVER ERROR


As mentioned a 5th call would move back to the beginning of the list:

200 OK
{ "weather" : "sunny", "celsius" : 32 }
